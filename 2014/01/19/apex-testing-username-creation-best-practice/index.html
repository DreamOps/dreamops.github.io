Apex Testing Username Creation Best Practice

[Luke Freeland] One common testing need is security testing. When a given user goes to Page X, they can only access and perform Y actions. Salesforce’s Apex Testing capabilities makes this painless. Through one or more test methods, users records are temporarily created and then verified that they can or cannot do the expected actions. This allows implementors to verify that their security model is correct or that it needs changing. – Read more at NimbleUser
